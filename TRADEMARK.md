# LogosLang Trademark Policy

_Last updated: 2026-06-17 · Status: early / bootstrap — this policy will evolve as the project matures._

## Why this file exists

The **LogosLang source code is free and open** under the Apache License 2.0 (see [`LICENSE`](./LICENSE)). You can read, fork, modify, build on, and redistribute it on those terms.

The **name "LogosLang" is a separate matter from the code.** A name is a trademark: it tells people *which* language they are getting and that it behaves the way the canonical project says it does. Open-source licenses deliberately do **not** grant trademark rights — Apache-2.0 states this in Section 6 ("This License does not grant permission to use the trade names, trademarks, service marks, or product names of the Licensor"). This file is the separate, permissive policy for the **name**.

The single goal is to **keep one unified LogosLang**: anyone may fork the code, but the *name* should always point to one thing, so that "LogosLang" means the same language to everyone. Within that goal, we want the mark used as freely as possible.

## What is claimed

- **"LogosLang"** — the word mark (in any capitalization; the canonical spelling is `LogosLang`, one word).
- The **LogosLang logo / wordmark**, once published (an official `brand/` directory will hold the assets and their terms).

**Not claimed: the bare word "Logos".** The project does not assert "Logos" or "Logos Language" as a trademark for this language; where those words appear they are descriptive. Please refer to the project as **LogosLang**.

## Uses that need no permission

You may, without asking:

- **Refer to LogosLang** truthfully — in articles, talks, tutorials, courses, books, research, blog posts, and discussion.
- **Say your work targets or supports it** when true — "a library for LogosLang", "written in LogosLang", "compatible with LogosLang", "LogosLang bindings for X".
- **Name community projects descriptively**, in a way that does not look official — prefer forms like `awesome-logoslang`, `logoslang-mode`, or "<Name>, a package manager for LogosLang". Avoid a name a reasonable person would read as *the* official project.
- **Redistribute the official software unmodified** under the name LogosLang.
- **Teach, benchmark, and critique** the language.

Rule of thumb: **tell the truth, and don't imply you are us.**

## Uses that need permission

Please ask first (or pick a different name) for:

- Naming a **modified or forked** compiler, interpreter, toolchain, or distribution "LogosLang" or anything confusingly similar (e.g. "LogosLang+", "LogosLang Pro", "RealLogosLang"). Forks must carry a **distinct name** (see *Compatibility* below).
- Using "LogosLang" in the name of a **company, product, app, paid service, or a GitHub org / domain / social account** in a way that suggests it is official.
- Using or modifying the **logo**.
- **Merchandise**, events, or anything implying **endorsement, affiliation, or sponsorship**.

## Compatibility — the one rule that keeps the ecosystem together

You may present your build as **"LogosLang"** only if it is the **unmodified** official software, or — once a conformance suite exists — it **passes the official LogosLang conformance tests**. This is what keeps the name meaningful: "LogosLang" code should run the same way everywhere.

If your version changes the language, or predates / fails conformance, give it **its own name**. You are welcome (and encouraged) to describe it accurately as **"a fork of LogosLang"**, **"based on LogosLang"**, or **"LogosLang-compatible"** where that is true.

## Attribution

On first or prominent use, please write **LogosLang™** (switch to **®** only after registration). Keep the camelCase spelling.

## Asking, and reporting misuse

Open an issue in the canonical LogosLang repository, or contact **thobknu@gmail.com**. If you see the name used in a way that confuses people about what is official, please let us know.

## Reservation of rights & notes

This policy may change as the project matures (governance, a published logo, a conformance suite, formal registration). It does not reduce any rights granted by Apache-2.0 over the **code**; it governs only the **name and logo**. Nothing here is legal advice.
